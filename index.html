What is Majordodo?

Majordodo is a replicated service to build complex multi-tenant computing grids. You can view it as a distributed threadpool where tasks are submitted by users and a network of brokers schedules execution on a large set of machines, caring about failures, recovery and priority-based resources assignment. Tasks are implemented (in version 1.0) in Java and can be submitted and monitored using a simple HTTP REST JSON API.

Multi-Tenancy

Majordodo manages resources for each tenant and guarantees resource access isolation and fair scheduling. User's execution priority is dynamic and can be modified at runtime for each and every task, even for the ones that have already been submitted to the scheduler.

Resiliency

Majordodo leverages Apache Zookeeper and Apache Bookkeeper to build a fully reaplicated, shared-nothing, broker architecture without any single point of failure. Workers can be run on comodity-hardware, virtual machines or dynamic provisioned cloud resources which can fail at any time without service interruption.

Scalability

Majordodo is designed to manage tasks for thousands of hosts. It is simple to add and remove hosts and resources with dynamic reconfiguration and adaptative tasks scheduling.
